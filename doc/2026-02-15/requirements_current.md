**프로젝트 명**: AI-GYM (AI 기반 공학 역량 강화 플랫폼)  
**버전**: 1.0 (Current Implementation)

---

## 1. 개요 (Introduction)
본 문서는 AI-GYM 시스템의 **현재 구현된** 핵심 기능 및 구조를 정의합니다.

---

## 2. 기술 스택 (Technical Stack)
- **Frontend**: Vue.js 3, Vite, Tailwind CSS, Pinia, Axios, Lucide Icons
- **Backend**: Python 3.x, Django REST Framework (DRF)
- **Database**: PostgreSQL / SQLite (환경별 상이), JSONField 활용
- **AI**: 나노바나나(아바타 생성), LLM 기반 의사코드 및 아키텍처 평가

---

## 3. 기능적 요구 사항 (Implemented Features)

### 3.1 사용자 관리 (User Management)
- **F-1.1 회원가입**:
    - **통합 가입**: 이메일, 성명, 생년월일, 닉네임 입력을 통한 계정 생성.
    - **유효성 검사**: 이메일 중복 체크, 비밀번호 8자 이상 검증.
- **F-1.2 로그인 및 세션**:
    - **이메일 로그인**: 이메일/비밀번호 기반 인증 및 Django 세션/세션쿠키 관리.
    - **로그아웃**: 세션 종료 및 서버 측 세션 쿠키 삭제.
    - **상태 유지**: 새로고침 시 세션 확인을 통한 사용자 인증 상태 복구.
- **F-1.3 프로필 관리**:
    - **개인 정보 수정**: 닉네임, 아바타, 직무 역할(Job Role), 관심 분야(Interests) 수정 기능.
    - **프로필 요약**: 현재 등급, 누적 점수 및 활동 상태 대시보드 표시.

### 3.2 아바타 시스템 (Nano Banana)
- **F-2.1 AI 아바타 생성**: 사용자 가입/수정 시 프롬프트(스타일)와 시드를 기반으로 고유 아바타 이미지 생성.
- **F-2.2 아바타 장착**: 생성된 이미지 중 대표 아바타를 설정하여 대시보드 및 리더보드에 노출.

### 3.3 연습 및 평가 시스템 (Practice & Evaluation)
- **F-3.1 데이터 전처리 연습 (Pseudocode)**:
    - **의사코드 입력**: 자연어 기반 데이터 전처리 로직 작성 및 제출.
    - **자동 평가**: 5차원 지표(설계력, 정합성, 구현력, 예외처리, 추상화) 기반 AI 채점 및 피드백.
    - **힌트 시스템 (Duck Hint)**: 실시간 키워드 감지를 통한 논리적 오류 가이드 제공.
- **F-3.2 시스템 아키텍처 연습 (Architecture)**:
    - **컴포넌트 설계**: GCP 주요 서비스를 드래그 앤 드롭으로 배치하고 연결 관계 설정.
    - **설계 평가**: 아키텍처의 적절성을 AI가 분석하여 평가 리포트 생성.

### 3.4 게이미피케이션 (Gamification)
- **F-4.1 포인트 및 등급**: 연습 해결 시 점수 획득 및 누적 점수에 따른 브론즈~엔지니어 등급 부여.
- **F-4.2 사용자 리더보드**: 전체 사용자 순위 및 등급별 분포 확인 기능.

---

## 4. 비기능적 요구 사항
- **NF-1.1 보안**: 비밀번호 단방향 암호화(PBKDF2) 저장 및 Django CSRF 보호 적용.
- **NF-1.2 확장성**: 연습 유닛(Unit)을 JSON 설정만으로 동적 추가 가능한 DB 구조.
