## 1. 구현 예정 기능 (To-Be Implemented)

### 1.1 사용자 관리 고도화
- **소셜 로그인 (OAuth 2.0)**: Google, GitHub 계정을 이용한 간편 로그인 및 회원가입 연동.
- **상세 보안 정책**:
    - 최근 사용한 비밀번호(직전 3개) 재사용 제한 기능.
    - 장기 미접속(1년 이상) 계정의 자동 휴면 전환 및 안내 시스템.
- **계정 보안 강화**: 이메일 기반 비밀번호 초기화 링크 발송 및 2차 인증(선택 사항).

### 1.2 실시간 협업 모드 (Roadmap)
- **실시간 캔버스 공유**: WebSocket 기술을 활용하여 다수의 사용자가 동시에 아키텍처를 설계.
- **공유 커서 및 프레젠스**: 협업 중인 팀원의 마우스 위치와 실시간 상호작용 표시.
- **편집 이력 관리**: 협업 과정에서의 수정 사항 추적 및 커밋 로그 작성.

### 1.3 어드민/운영 기능
- **관리자 대시보드**: 전체 사용자 활동 로그 분석 및 유닛별 통계 리포트 조회.
- **공지사항 및 푸시 알림**: 시스템 점검 또는 새로운 콘텐츠 업데이트 실시간 알림.

## 2. 검토 필요 사항 (Needs Review)

### 2.1 성능 및 인프라
- **AI 응답 스트리밍**: LLM 평가 결과 생성 시 대기 시간을 줄이기 위한 실시간 텍스트 스트리밍 방식 도입 검토.
- **인프라 통합 모니터링**: GCP Cloud Logging 및 Monitoring을 활용한 API 성능 추적.

### 2.2 사용자 경험 (UX)
- **모바일 앱(WebView) 지원**: 모바일 환경에서의 아키텍처 드래그 앤 드롭 UI 최적화 방안 검토.
- **동료 피드백 시스템**: 사용자가 작성한 아키텍처나 의사코드에 대해 동료들이 리뷰하고 의견을 남기는 소셜 기능 도입 검토.
