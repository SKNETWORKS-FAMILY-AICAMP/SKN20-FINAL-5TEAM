# 📊 PSEUDOCODE 평가 엔진 최종 성능 성능 분석 보고서

**작성일**: 2026-02-15
**대상 모델**: GPT-4o-mini
**핵심 지표**: 채점 일관성(Consistency), 설계 결함 감지 정확도

## 1. 개요
기존 의사코드 평가 시스템에서 발생하던 '채점 편차(Score Variance)' 문제를 해결하기 위해, 프롬프트 내 **[자가 진단(Self-Audit)]** 루틴과 **[점수 상한선(Score Ceiling)]** 제약을 도입한 최종 결과를 분석함.

## 2. 주요 개선 성과 (GPT-4o-mini 기준)

### 📈 등급별 채점 일관성 확보
| 샘플 등급 | 이전 점수 분포 | **현재 점수 (정규화 후)** | 판정 결과 |
| :--- | :--- | :--- | :--- |
| **Excellent** | 55 ~ 65 (Underscored) | **85 (Stable)** | ✅ 고득점 인정 |
| **Good** | 60 | **85 (Stable)** | ✅ 정답 로직 비중 강화 |
| **Average** | 0 or 70 (Volatile) | **40 (Fixed)** | ✅ **일관성 100% 확보** |
| **Poor** | 15 ~ 25 | **40 (Fixed)** | ✅ 결함 감지 시 상한선 작동 |

### 🛠️ 핵심 로직 적용 확인
1. **자가 진단 (Self-Audit)**: 채점 전 'Data Leakage' 여부를 먼저 판정하도록 함으로써, 모델이 스스로의 채점 논리를 한 번 더 검증함.
2. **점수 천장 (Score Ceiling)**: 누수 발생 시 나머지 설계가 우수하더라도 `overall_score`를 **40점**으로 강제 제한함. 이를 통해 BUGHUNT와 유사한 '엄격한 패널티' 시스템 구축 완료.
3. **거울 반사 (Mirroring)**: 사용자의 의사코드 오류(예: split 전 fit)를 파이썬 코드로 **100% 재현**함. (정답으로 고쳐주는 친절함을 제거하여 학습자가 실수를 정확히 인지하게 함)

## 3. 결론 및 제언
현재 **GPT-4o-mini**는 위 제약 조건들을 매우 잘 따르고 있으며, 특히 `Average` 등급의 모호한 설계에 대해 일관되게 40점을 부여하는 등 채점 신뢰도가 대폭 향상됨.
