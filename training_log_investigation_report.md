# 훈련 기록 제목 및 데이터 정합성 조사 보고서

사용자가 제보한 훈련 기록 제목 중복 현상 및 각 유닛(Unit 01, 02, 03)의 데이터 상태를 조사한 결과입니다.

## 1. 조사 결과 요약

| 유닛 | 상태 | 주요 현상 및 원인 |
| :--- | :--- | :--- |
| **Unit 01** | **정상 (중복 풀이)** | DB상의 문제 제목은 고유하지만, 사용자가 **동일한 문제를 여러 번 풀이**하여 여러 개의 기록이 표시됨. (예: '맛있는 라면 끓이기' 3회 풀이) |
| **Unit 02** | **데이터 누락** | `Practice` 항목(Debug Practice)은 존재하나, 하위 문제(`PracticeDetail`) 데이터가 DB에 한 건도 존재하지 않음. |
| **Unit 03** | **DB 데이터 중복** | 동일한 제목의 문제가 **두 가지 ID 포맷(`unit0301` vs `unit03_01`)으로 이중 등록**되어 있어 혼선이 발생할 수 있음. |

## 2. 유닛별 상세 분석

### Unit 01 (Pseudo Practice)
- **현상**: '맛있는 라면 끓이기' 제목이 세 번 나타남 (점수: 40, 1575, 40).
- **분석**: DB 확인 결과 `unit01_01`(라면 끓이기)은 한 개만 존재함. 즉, 사용자가 해당 문제를 세 번 완료했기 때문에 각 완료 시점의 기록이 모두 노출되는 것입니다.
- **영향**: 기능적 오류는 아니나, 기록이 많아질 경우 가독성이 떨어질 수 있습니다.

### Unit 02 (Debug Practice)
- **현상**: 유닛 선택은 가능하나 기록이 전혀 나타나지 않음.
- **분석**: `PracticeDetail` 테이블에 `practice_id='unit02'`인 데이터가 없습니다. 초기 데이터(Fixture) 로드 시 누락되었거나 ID가 다르게 정의되었을 가능성이 큽니다.
- **영향**: 사용자가 Unit 02 문제를 풀더라도 기록에 남지 않거나, 아예 풀 문제가 없을 수 있습니다.

### Unit 03 (System Practice)
- **현상**: 제목은 같으나 ID가 다른 데이터가 쌍으로 존재함.
- **분석**: 
    - 예: `unit0301` (ID) vs `unit03_01` (ID) -> 둘 다 "Amazon 판매 랭킹 시스템"
    - 데이터 이관 또는 Fixture 중복 로드 과정에서 ID 명명 규칙이 혼용되어 발생한 문제입니다.
- **영향**: 기록창에서 동일한 제목이 서로 다른 문제인 것처럼 이중으로 나타나 사용자에게 혼란을 줍니다.

## 3. 해결 방안 제안

1. **백엔드 로직 수정 (가독성 개선)**: `UserAnswersView`에서 동일 문제에 대한 여러 풀이 기록이 있을 경우, **최신 기록만 보여주거나 그룹화**하여 표시하도록 수정.
2. **데이터 클리닝 (Unit 03)**: Unit 03의 중복된 `PracticeDetail` 데이터를 정리하여 단일 ID 체계로 통일.
3. **데이터 보충 (Unit 02)**: 누락된 Unit 02의 연습 문제 데이터를 DB에 추가.
