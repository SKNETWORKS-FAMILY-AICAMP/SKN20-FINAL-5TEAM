## 미달 원인 분석

### 구분력 미달 (28.9pt — 목표 30pt에 1.1pt 차이)

4개 문제의 very_poor 평균 점수가 42.9점으로, excellent 평균(71.8점)과 충분한 격차를 확보하지 못하고 있습니다. 문제별 격차를 보면 jr_004 이미지 피드의 very_poor 평균이 50.4점으로 가장 높아 전체 평균을 끌어올리는 주요 원인입니다.

| 문제 | excellent | very_poor | 격차 |
|---|---|---|---|
| jr_001 URL Shortener | 67.6 | 41.4 | 26.2pt |
| jr_002 Pastebin | 74.0 | 45.6 | 28.4pt |

jr_004 very_poor 샘플(50.4점)이 문제의 핵심입니다. 이미지 피드 문제에서 very_poor 수준의 설계도 "CDN을 쓰겠습니다", "Object Storage를 사용하겠습니다"라는 막연한 언급만으로 6개 축에서 각각 fair(55~71점) 등급을 받아 점수가 높게 형성됩니다.

### 축별 구분력의 구조적 한계

6개 Well-Architected 축 중 성능 최적화(17.3pt)와 지속가능성(21.4pt)은 어떤 아키텍처 설계 문제에서도 very_poor 수준의 답변이 일반적인 표현으로 점수를 얻기 쉬운 항목입니다. 이 두 축이 전체 구분력 평균을 낮추는 구조적 원인입니다.
